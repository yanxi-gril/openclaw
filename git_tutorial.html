📚 Git 使用详解

版本控制系统的完整命令参考指南

🎯 什么是 Git？

Git 是由 Linus Torvalds 于 2005 年创建的分布式版本控制系统。它用于追踪文件变化、协调多人协作开发，是当今最流行的版本控制工具。


核心概念：

  • 工作区：你实际编辑文件的目录
  • 暂存区 (Stage)：准备提交的文件集合
  • 仓库 (Repository)：保存所有历史版本的地方
  • 分支 (Branch)：并行开发的独立线

一、初始化与配置

🔧 基础配置命令

git init 常用

初始化一个新的 Git 仓库

详细说明：

在当前目录创建一个隐藏的 .git 文件夹，用于存储所有版本控制信息。如果目录已是 Git 仓库，会提示 "Reinitialized existing Git repository"。

示例：初始化项目
$ mkdir my-project
$ cd my-project
$ git init
Initialized empty Git repository in /path/to/my-project/.git/
git clone <url> 常用

克隆远程仓库到本地

详细说明：

从远程地址复制整个仓库（包括所有分支和历史记录）到本地。会自动创建同名文件夹。

示例：克隆仓库
$ git clone https://github.com/user/repo.git
$ git clone -b main --single-branch https://github.com/user/repo.git
# 只克隆 main 分支
$ git clone --depth 1 https://github.com/user/repo.git
# 浅克隆，只获取最新一次提交
git config 常用

配置 Git 设置

详细说明：

用于设置用户信息、别名等配置。分为三个级别：--global（全局）、--local（当前仓库）、--system（系统级）。

示例：配置用户信息
$ git config --global user.name "Your Name"
$ git config --global user.email "your@email.com"
$ git config --global core.editor "vim"
$ git config --global alias.co checkout
$ git config --list # 查看所有配置

二、基本工作流

📝 文件状态管理

git status 常用

查看仓库当前状态

详细说明：

显示哪些文件被修改、哪些已添加到暂存区、哪些未跟踪。是开发过程中最常用的命令之一。

示例：查看状态
$ git status
# 输出示例：
# On branch main
# Changes not staged for commit:
# (use \"git add \" to update what will be committed)
# (use \"git restore \" to discard changes)
# modified: README.md
# Untracked files:
# (use \"git add \" to include in what will be committed)
# new_file.py
git add 常用

将文件添加到暂存区

详细说明：

把工作区的修改放入暂存区，准备提交。可以添加单个文件、多个文件或整个目录。

示例：添加文件
$ git add README.md # 添加单个文件
$ git add *.py # 添加所有 .py 文件
$ git add src/ # 添加整个目录
$ git add . # 添加所有修改的文件
$ git add -A # 添加所有文件（包括删除）
$ git add -p # 交互式添加，可选择文件的部分修改
git commit 常用

提交暂存区的修改

详细说明：

将暂存区的修改永久保存到仓库。每次提交都会生成唯一的 SHA-1 哈希值。

示例：提交修改
$ git commit -m "添加用户登录功能"
$ git commit -m "修复 bug" -m "详细描述在第二行"
$ git commit # 打开编辑器写提交信息
$ git commit --amend # 修改上一次提交
$ git commit --allow-empty -m "触发 CI" # 空提交

🔄 撤销与恢复

git reset 常用

重置 HEAD 和暂存区到指定状态

详细说明：

--mixed（默认）：移动 HEAD，保留工作区修改
--soft：只移动 HEAD，保留暂存区和工作区
--hard：全部重置，丢失所有未提交修改

示例：重置操作
$ git reset HEAD~1 # 回退一次提交（保留修改）
$ git reset --soft HEAD~2 # 回退两次（保留暂存）
$ git reset --hard # 彻底回退（危险！）
$ git reset filename # 从暂存区移除文件
git restore 新命令

恢复文件到指定状态（Git 2.23+）

详细说明：

新版 Git 推荐的撤销命令，比 checkout 更语义化。可用于恢复工作区或暂存区的文件。

示例：恢复文件
$ git restore filename # 从暂存区恢复工作区
$ git restore --staged filename # 从 HEAD 恢复暂存区
$ git restore --source=HEAD~1 filename # 从历史版本恢复
git checkout 常用

切换分支或恢复文件

详细说明：

主要用于切换分支。也可用于从暂存区恢复工作区的文件（但推荐用 restore）。

示例：切换与恢复
$ git checkout feature # 切换到 feature 分支
$ git checkout -b new-feature # 创建并切换新分支
$ git checkout HEAD -- file # 恢复文件到 HEAD 版本
$ git checkout -- file # 丢弃工作区修改

三、分支操作

🌿 分支管理

git branch 常用

列出、创建或删除分支

详细说明：

不带参数时列出所有本地分支。配合参数可创建、删除、重命名分支。

示例：分支操作
$ git branch # 列出所有分支
$ git branch -a # 列出所有分支（含远程）
$ git branch feature # 创建新分支
$ git branch -d feature # 删除已合并分支
$ git branch -D feature # 强制删除
$ git branch -m old new # 重命名分支
$ git branch --contains HEAD # 查看包含当前提交的分支
git merge 常用

合并分支

详细说明：

将指定分支的改动合并到当前分支。Git 会尝试自动合并，冲突时需要手动解决。

示例：合并分支
$ git merge feature # 合并 feature 分支
$ git merge --no-ff feature # 禁用快进，保留合并记录
$ git merge --squash feature # 压缩合并，单条提交
# 解决冲突后：
$ git add . && git commit # 完成合并提交
git rebase 常用

变基（重写提交历史）

详细说明：

将当前分支的提交"移动"到另一个分支顶部，产生线性的历史。比 merge 更整洁，但会改变提交哈希。

示例：变基操作
$ git rebase main # 将当前分支变基到 main
$ git rebase -i HEAD~3 # 交互式变基最近 3 次提交
# 交互模式下可：
# pick - 保留提交
# squash - 合并到前一个提交
# edit - 修改提交
# drop - 删除提交
$ git rebase --abort # 中止变基
$ git rebase --continue # 继续变基（解决冲突后）
git cherry-pick

挑选特定提交应用到当前分支

详细说明：

从任意分支提取一个或多个提交的改动，应用到当前分支。适合选择性合并。

示例：挑选提交
$ git cherry-pick abc1234 # 应用单个提交
$ git cherry-pick abc1234 def5678 # 应用多个提交
$ git cherry-pick abc1234^! # 应用提交及其祖先
$ git cherry-pick --no-commit abc1234 # 不自动提交

四、远程操作

🌐 远程仓库管理

git remote 常用

管理远程仓库地址

详细说明：

用于查看、添加、删除远程仓库。克隆的仓库默认有名为 origin 的远程。

示例：远程管理
$ git remote -v # 查看所有远程
$ git remote add origin https://github.com/user/repo.git # 添加远程
$ git remote rename origin upstream # 重命名
$ git remote remove origin # 删除远程
$ git remote set-url origin https://new-url.com # 修改地址
git fetch 常用

获取远程更新

详细说明：

从远程仓库下载新的分支、提交和文件，但不自动合并。是 merge 和 pull 的前置操作。

示例：获取更新
$ git fetch origin # 获取 origin 所有更新
$ git fetch origin main # 只获取 main 分支
$ git fetch --all # 获取所有远程
$ git fetch origin --prune # 获取并清理已删除分支
git pull 常用

获取并合并远程更新

详细说明：

相当于 fetch + merge。获取远程更新并立即合并到当前分支。

示例：拉取更新
$ git pull # 拉取当前分支
$ git pull origin main # 拉取 origin/main
$ git pull --rebase # 用 rebase 方式拉取
$ git pull origin feature --no-commit # 不自动提交
git push 常用

推送本地修改到远程

详细说明：

上传本地分支的提交到远程仓库。首次推送分支需要设置上游。

示例：推送修改
$ git push # 推送到上游
$ git push origin main # 推送到 origin/main
$ git push -u origin feature # 推送并设置上游
$ git push origin :feature # 删除远程 feature 分支
$ git push --force # 强制推送（危险！）
$ git push --force-with-lease # 安全的强制推送

五、查看与对比

🔍 日志与差异

git log 常用

查看提交历史

详细说明：

显示提交历史记录。支持多种格式和过滤选项。

示例：查看日志
$ git log # 默认格式
$ git log --oneline # 简洁单行格式
$ git log --graph # 图形化分支
$ git log --graph --oneline --all # 图形化所有分支
$ git log -p # 显示每次提交的改动
$ git log --stat # 显示文件统计
$ git log --since="2 weeks ago" # 时间过滤
$ git log author:"John" # 作者过滤
$ git log --grep="fix" # 搜索提交信息
$ git log src/ # 只查看某个路径的历史
git diff 常用

查看差异

详细说明：

显示不同状态之间的文件差异。是理解改动内容的重要工具。

示例：查看差异
$ git diff # 工作区 vs 暂存区
$ git diff --staged # 暂存区 vs HEAD
$ git diff HEAD # 工作区 vs HEAD
$ git diff HEAD~1 # 与上次提交对比
$ git diff main feature # 分支间对比
$ git diff abc123 def456 # 两个提交对比
$ git diff --word-diff # 按单词显示差异
$ git diff --color-words # 彩色单词差异
git show

显示提交详情

详细说明：

显示指定提交的详细信息，包括提交信息和文件改动。

示例：查看提交
$ git show # 显示当前 HEAD
$ git show abc123 # 显示指定提交
$ git show HEAD~2 # 显示上上次提交
$ git show HEAD:filename # 显示文件在 HEAD 的内容
$ git show --stat # 显示统计信息
git blame

逐行显示文件修改者

详细说明：

显示文件中每一行最后修改的提交、作者和时间。用于追踪代码来源。

示例：代码溯源
$ git blame filename # 查看文件溯源
$ git blame -L 10,20 filename # 查看指定行范围
$ git blame --line-porcelain # 机器可读格式
$ git blame --ignore-rev-heads # 忽略合并提交

六、高级功能

🚀 高级操作

git stash 常用

暂存当前修改

详细说明：

将工作区的修改临时保存，便于切换分支。可随时恢复。

示例：暂存操作
$ git stash # 暂存修改
$ git stash save "描述信息" # 带描述的暂存
$ git stash list # 列出所有暂存
$ git stash pop # 恢复并删除最新暂存
$ git stash apply # 恢复但保留暂存
$ git stash apply stash@{1} # 恢复指定暂存
$ git stash drop # 删除最新暂存
$ git stash clear # 删除所有暂存
git tag 常用

创建和管理标签

详细说明：

为特定提交添加标记，常用于版本发布。分为轻量标签和附注标签。

示例：标签操作
$ git tag v1.0.0 # 轻量标签
$ git tag -a v1.0.0 -m "版本说明" # 附注标签
$ git tag # 列出所有标签
$ git tag -l "v1.*" # 模糊匹配
$ git show v1.0.0 # 查看标签信息
$ git push origin --tags # 推送所有标签
$ git push origin tag v1.0.0 # 推送单个标签
$ git tag -d v1.0.0 # 删除本地标签
git reflog 常用

查看 HEAD 操作历史

详细说明：

记录 HEAD 的所有移动操作，包括被 reset、rebase 等操作覆盖的提交。是恢复误操作的神器。

示例：查看操作日志
$ git reflog # 查看 HEAD 历史
$ git reflog show main # 查看分支历史
$ git reflog expire --expire=now -- --branches # 清理
# 误删除后恢复：
$ git reflog # 找到丢失的提交
$ git reset --hard HEAD@{1} # 恢复到之前状态
git bisect

二分查找引入 bug 的提交

详细说明：

通过二分查找快速定位引入问题的提交。需要能够判断某版本是否有问题。

示例：二分查找
$ git bisect start # 开始二分
$ git bisect bad # 标记当前为坏版本
$ git bisect good v1.0 # 标记已知好版本
# Git 会切换到中间版本，测试后：
$ git bisect good # 如果没问题
$ git bisect bad # 如果有问题
# 找到后：
$ git bisect reset # 结束二分
git submodule

管理子模块

详细说明：

将一个 Git 仓库作为另一个仓库的子目录。用于代码复用和依赖管理。

示例：子模块操作
$ git submodule add https://github.com/user/lib.git path/to/lib # 添加
$ git submodule update --init # 初始化子模块
$ git submodule update --recursive # 递归更新
$ git submodule status # 查看状态
$ git submodule foreach git pull # 对所有子模块执行命令
git revert

反向应用提交（安全撤销）

详细说明：

创建一个新的提交，其改动与指定提交相反。不会改变历史，适合已推送的提交。

示例：反向提交
$ git revert abc123 # 反向单个提交
$ git revert abc123^..abc123 # 等价于上面
$ git revert -n abc123 # 不自动提交
$ git revert HEAD~3..HEAD # 反向多个提交
$ git revert --no-commit HEAD~3..HEAD # 合并后一起提交

七、实用技巧

💡 常用别名配置

推荐添加到 ~/.gitconfig
[alias]
    st = status
    co = checkout
    br = branch
    ci = commit
    di = diff
    unstage = reset HEAD
    amend = commit --amend
    last = log -1 HEAD
    graph = log --graph --oneline --all
    new = branch -a --contains HEAD

📋 常用工作流

功能开发工作流
# 1. 同步最新代码
git fetch origin && git checkout main && git pull

# 2. 创建功能分支
git checkout -b feature/new-feature

# 3. 开发并提交
git add . && git commit -m "feat: add new feature"

# 4. 同步主分支（可选）
git fetch origin && git rebase origin/main

# 5. 推送并创建 PR
git push -u origin feature/new-feature

# 6. 合并后删除分支
git checkout main && git pull
git branch -d feature/new-feature
