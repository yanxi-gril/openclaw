Agent 区别与联系

主 agent、@explore、@general 三大角色详解

🎭 三大 Agent 简介

👑 主 agent（我）

  • 定位：全能型，直接响应用户请求
  • 能力：拥有所有工具的完整权限
  • 任务：文件操作、代码修改、运行命令等日常任务
  • 特色：可以调用子 agent 来辅助完成特定任务

🔍 @explore 子 agent

  • 定位：代码探索专用专家
  • 核心功能：
    • 快速文件搜索 - 通过 glob 模式查找文件（如 **/*.py）
    • 关键词代码搜索 - 在代码库中搜索特定内容（如函数名、类定义）
    • 回答代码库问题 - 解释代码结构、模块关系等
  • 工具权限：Glob、Grep、Read（探索相关工具）
  • 适用场景：
    • 需要了解大型代码库结构时
    • 快速定位某个功能实现的代码位置
    • 查找使用了某个类/函数的地方
    • 探索陌生项目的组织方式
  • 详细程度：支持 quick（基础）、medium（中等）、very thorough（全面）三级
  • 不适用场景：精确查找已知路径的单个文件、在特定 2-3 个文件中搜索、需要修改代码的任务

⚙️ @general 子 agent

  • 定位：通用型复杂任务处理专家
  • 核心能力：
    • 处理复杂的多步骤任务
    • 研究需要综合分析的问题
    • 自主规划和执行任务
  • 工具权限：全面的工具集，支持多工具协同
  • 适用场景：
    • 需要同时使用多个工具的复杂查询
    • 跨文件/跨模块的分析任务
    • 需要自主规划执行步骤的任务
    • 开放性研究问题
  • 与@explore 的区别：@explore 专注代码探索；@general 专注复杂任务处理

📊 核心区别对比

维度 主 agent @explore @general
定位 全能型 探索专用 复杂任务专用
独立性 完全独立 独立运行 独立运行
返回结果 给用户 给主 agent 给主 agent
工具权限 完整 探索相关 综合型

🔗 三者的联系

协作模式：

  1. 主 agent 接收用户请求
  2. 根据任务类型，可委派给合适的子 agent
  3. 子 agent 独立运行，完成研究/探索
  4. 子 agent 将结果返回给主 agent
  5. 主 agent 整合所有信息，最终回复用户

💡 简单比喻

主 agent = "前台经理" —— 直接服务客户（用户）

@explore = "档案管理员" —— 专门负责查找资料

@general = "项目专员" —— 处理复杂项目
